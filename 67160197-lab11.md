## ส่วนที่ 1: JSX Syntax & JSX Rules

## คำถาม 1.1: จงอธิบายความแตกต่างระหว่าง class และ className ใน JSX พร้อมทำให้ตัวอย่างที่แสดงเหตุผลว่าทำไม React ต้องการใช้ className

แนวทางการตอบคำถาม: อธิบายให้ชัดเจน มีตัวอย่างถูกต้อง และอธิบายเหตุผล

ตัวอย่างคำตอบ:

JSX ต้องใช้ className แทน class เพราะ:

1. class เป็นคำสงวน (reserved keyword) ใน JavaScript
2. JSX แปลง HTML attribute ให้เป็น JavaScript properties
3. ตัวอย่าง:
   - ถูก: <div className="container">...</div>
   - ผิด: <div class="container">...</div> (error)

ใน JavaScript ทั่วไป, className ใช้เพื่อหลีกเลี่ยงการชนกับคำสงวน class

## คำถาม 1.2: จงเขียน Component ที่ใช้ JSX rules ให้ถูกต้องตามตัวอย่าง

// ต้องเขียนให้ถูกต้อง (แก้ไขโค้ดที่ผิด)

            function ProfileCard() {
              const userName = "สมชาย";
              const age = 25;

              return (
                <div class="card">
                  <h3>ชื่อ: userName</h3>
                  <p>อายุ: age</p>
                  <img src="avatar.png">
                  <button>คลิกที่นี่</button>
                </div>
              );
            }

ให้แสดง Component ที่แก้ไขแล้ว และอธิบายข้อผิดพลาดแต่ละข้อ

            function ProfileCard() {
              const userName = "สมชาย";
              const age = 25;

              return (
                <div className="card">
                  <h3>ชื่อ: {userName}</h3>
                  <p>อายุ: {age}</p>
                  <img src="avatar.png" alt="avatar" />
                  <button>คลิกที่นี่</button>
                </div>
              );
            }

- สิ่งที่ทำการแก้ไขไปนั้นมี

* <div className="card"> เปลี่ยนเพื่อป้องกัน Error เป็น Classname แทนเลย
* <img src="avatar.png" alt="avatar" /> ต้องปิดแท็กด้วย /> และเพิ่ม alt
* <p>อายุ: {age}</p> ต้องใช้งาน {age} แทนเพื่อแสดงค่าตัวแปร
* <h3>ชื่อ: {userName}</h3> ต้องใช้งาน {userName} แทนเพื่อแสดงค่าตัวแปร

## คำถาม 2.2: จงเปรียบเทียบวิธีการเขียน Function Component แบบ regular function และ arrow function พร้อมกล่าวถึง ข้อดีข้อเสีย

ตัวอย่างการเขียน

            // Regular function
            function Greeting() {
              return <h1>Hello</h1>;
            }

            // Arrow function
            const Greeting = () => <h1>Hello</h1>;

## ข้อดีข้อเสีย

- Regular function สะดวกต่อการใช้งานอ่านง่าย, ใช้ this ได้ใน class-based context

- Arrow function โค้ดดูเรียลเข้าใจได้กระชับ, ไม่มี this binding, นิยมใช้ใน modern React เพราะเขียนสั้นกว่า

## ส่วนที่ 3: Props

## คำถาม 3.1: จงสร้าง Component ชื่อ ProductCard ที่รับ Props: name, price, inStock พร้อมอธิบายวิธีการส่ง Props จาก Parent Component

// ต้องสร้าง Component นี้:
// 1. ProductCard Component
// 2. แสดงวิธีการใช้ใน App.jsx (ส่ง Props)
// 3. อธิบายความแตกต่างระหว่าง string, number, boolean props

- Component
  function ProductCard({ name, price, inStock }) {
  return (
  <div className="product-card">
  <h3>{name}</h3>
  <p>ราคา: {price} บาท</p>
  <p>{inStock ? "มีสินค้า" : "หมดสินค้า"}</p>
  </div>
  );
  }
            // ใช้ใน App.jsx
            function App() {
              return (
                <div>
                  <ProductCard name="Laptop" price={25000} inStock={true} />
                  <ProductCard name="Mouse" price={500} inStock={false} />
                </div>
              );
            }
- การส่งข้อมูล

* ส่งจาก Parent Component ไปยัง Child Component เช่น name="Laptop" คือการส่งค่า "Laptop" ไปให้ ProductCard

- ความแตกต่างของชนิดข้อมูล Props

* String ต้องใส่เป็น "Laptop" (ต้องใส่เครื่องหมาย " ")
* Number ต้องใส่เป็น {25000} (ต้องใส่ใน {})
* Boolean ต้องใส่เป็น {true} หรือ {false} (ต้องใส่ใน {})

## คำถาม 3.2: Props destructuring คืออะไร? จงแสดงตัวอย่างเปรียบเทียบระหว่าง:

วิธีเก่า: function Card(props) { ... }
วิธีใหม่: function Card({ title, description }) { ... }
และอธิบายว่าทำไมวิธีใหม่ (destructuring) จึงเป็น best practice

- วิธีเก่า

            function Card(props) {
              return <h3>{props.title} - {props.description}</h3>;
            }

- วิธีใหม่

            function Card({ title, description }) {
              return <h3>{title} - {description}</h3>;
            }

- วิธีใหม่ทำไมถึงเป็น best practice

* อ่านง่ายขึ้นลดการที่ต้องเขียน props.title ซ้ำ ๆ
* ชัดเจนว่า Component ต้องการใช้ props อะไรบ้าง
* ลดความสับสนที่ต้องเสียเวลาในการ Component มี props เยอะ การ destructuring ทำให้โค้ดดูสะอาด

## ส่วนที่ 4: Lists & .map()

## คำถาม 4.1: จงอธิบายว่า .map() ทำงานอย่างไรในการแสดงรายการข้อมูล พร้อมตัวอย่างโค้ดที่แสดง:

การใช้ .map() เพื่อแสดง list ของ Component
เหตุผลว่าทำไมต้องใช้ key property
ข้อแตกต่างระหว่าง key={index} และ key={item.id}

- การใช้ .map()

* .map() เป็นฟังก์ชันของ Array ที่ใช้วนลูปและสร้าง Array ใหม่
* ใน React ใช้ .map() เพื่อสร้าง หลาย ๆ Component จากข้อมูลใน Array

- ตัวอย่าง

            const items = [{id:1, name:"A"}, {id:2, name:"B"}];

            function ItemList() {
              return (
                <ul>
                  {items.map(item => <li key={item.id}>{item.name}</li>)}
                </ul>
              );
            }

- เหตุผลที่ต้องใช้ key

* React ต้องการ key เพื่อรู้ว่า element ไหนเปลี่ยน, เพิ่ม, หรือลบ
* ถ้าไม่มี key React จะ render ใหม่ทั้ง list นั้นจะทำให้ performance แย่

- ความแตกต่าง

* key={index} ไม่ดี เพราะ index อาจเปลี่ยนถ้ามีการเพิ่ม/ลบข้อมูล
* key={item.id} ดี เพราะเป็น unique และ stable

## คำถาม 4.2: สร้าง Component ที่ใช้ .map() เพื่อแสดง:

รายชื่อนิสิต (array of objects ที่มี id, name, grade)
แสดงเป็นตาราง โดยใช้ .map() ในการสร้าง แต่ละแถว
อธิบายเหตุผลว่าทำไมต้องใช้ unique ID เป็น key

- ตัวอย่าง

const students = [
{ id: 1, name: "สมชาย", grade: "A" },
{ id: 2, name: "สมหญิง", grade: "B+" },
{ id: 3, name: "อนันต์", grade: "C" }
];

            function StudentTable() {
              return (
                <table>
                  <thead>
                    <tr>
                      <th>รหัส</th>
                      <th>ชื่อ</th>
                      <th>เกรด</th>
                    </tr>
                  </thead>
                  <tbody>
                    {students.map(student => (
                      <tr key={student.id}>
                        <td>{student.id}</td>
                        <td>{student.name}</td>
                        <td>{student.grade}</td>
                      </tr>
                    ))}
                  </tbody>
                </table>
              );
            }

- เหตุผลที่ต้องใช้งาน unique ID เป็น key

* ใช้ student.id เป็น key เพราะมัน unique มันจะทำการ React จะ track ได้ถูกต้อง
* ถ้าใช้ index เมื่อเราทำการแก้ไข เวลาเพิ่ม/ลบข้อมูล React อาจสับสนและ render ผิด

## ส่วนที่ 5: Best Practices & Naming Conventions

## คำถาม 5.1: จงอธิบายและยกตัวอย่างการตั้งชื่อในแต่ละประเภท ทั้งแบบที่ถูก และแบบที่ผิด:

Component Names - ตั้งชื่อแบบไหน?

- Component Names ใช้งาน PascalCase

* แบบถูก UserProfile
* แบบผิด user_profile

  Variable Names - ตั้งชื่อแบบไหน?

- Variable Names ใช้งาน camelCase

* แบบถูก userName
* แบบผิด User_Name

  Constants - ตั้งชื่อแบบไหน?

- Constants ใช้งาน UPPER_CASE

* แบบถูก MAX_USERS
* แบบผิด maxUsers

  File Names - ตั้งชื่อแบบไหน?

- File Names ใช้งาน PascalCase หรือ kebab-case

* แบบถูก UserProfile.jsx หรือ user-profile.jsx
* แบบผิด user_profile.jsx

## คำถาม 5.2: จากโค้ด Component ต่อไปนี้ให้บอก:

ส่วนที่ตั้งชื่อถูกต้อง
ส่วนที่ตั้งชื่อผิด (ถ้ามี)
แก้ไขโค้ดให้เป็น best practice

            function user_profile({ user_name, user_age, IS_PREMIUM }) {
              const user_data = {
                name: user_name,
                age: user_age,
                isPremium: IS_PREMIUM,
              };

              return (
                <div>
                  <h3>{user_data.name}</h3>
                  <p>{user_data.age} ปี</p>
                </div>
              );
            }

- โค้ดที่ถูกแก้ไขให้เป็น best practice แล้ว

            function UserProfile({ userName, userAge, isPremium }) {
              const userData = {
                name: userName,
                age: userAge,
                isPremium: isPremium,
              };

              return (
                <div>
                  <h3>{userData.name}</h3>
                  <p>{userData.age} ปี</p>
                  {userData.isPremium && <p>สมาชิกพรีเมียม</p>}
                </div>
              );
            }

- อะไรบ้างที่ได้รับการปรุงปรับ

* Props คือค่าที่ส่งจาก Parent เปลี่ยนเป็น Child

* ใช้ {} เพื่อแสดงค่าตัวแปรใน JSX

* .map() ใช้สร้าง list ของ Components และต้องมี key

* การตั้งชื่อควรใช้ PascalCase สำหรับ Component, camelCase สำหรับตัวแปร, UPPER_CASE สำหรับ constants
